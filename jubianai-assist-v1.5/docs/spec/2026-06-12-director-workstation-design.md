# AI 导演工作台 · 设计文档

## 概述

将 jubianai-assist 从图片匹配助手改造为 AI 导演工作台。三栏自由面板系统，蜘蛛侠纵横宇宙视觉风格，聚焦文本处理 + 资产管理的工作流。

---

## 一、布局架构

### 总体布局

```
┌──────────────────────────────────────────────────────────┐
│               🎬 AI 导演工作台 — 拖拽标题栏                │
├──────────┬───────────────────────────┬───────────────────┤
│  L1      │          C                │        R1         │
│  📂 Skill│      💬 交互区             │    🖼️ 资产池      │
│  栏      │                           │     栏            │
│  L2      │                           │    R2             │
│  L3      │                           │    R3             │
│          │                           │                   │
│          │                           │   点击图片→画图    │
├──────────┴───────────────────────────┴───────────────────┤
│              📋 子任务列表（底部面板）                     │
└──────────────────────────────────────────────────────────┘
```

### 面板特性

- 三栏为默认布局，每个栏是一个独立面板
- 每个面板可拖拽标题栏移动位置
- 每个面板右下角有拖拽手柄可自由调整大小
- 面板可叠放、可暂时折叠（最小化到边缘）
- 位置和大小保存在 chrome.storage.local，刷新保留
- 面板边框为不规则的漫画手绘风格线条

### 三栏说明

| 栏 | 默认宽度 | 内容 |
|:--:|:--------:|------|
| 左 Skill 栏 | 220px | 内置 skill 列表 + 用户上传 skill + 文件夹整理 + 删除 |
| 中 交互栏 | 1fr | LLM 对话/输出区，用户复制文本或生成 .txt |
| 右 资产栏 | 280px | 三类资产标签页 + 点击图片进入画图 + 文件夹整理 + 删除 |

### 底部面板

| 面板 | 说明 |
|:----|------|
| 子任务列表 | 显示当前处理出的子任务/镜头，可勾选保存为 .txt 或 .skill |

## 二、Skill 系统

### 内置 Skill

| Skill | 来源路径 | 说明 |
|:------|:---------|------|
| script-analyzer | `E:\cc wright\.claude\skills\script-analyzer\SKILL.md` | 剧本分析，提取角色/场景/道具 |
| shot-script-creator | `E:\cc wright\.claude\skills\shot-script-creator-by-sweet-v4.2\SKILL.md` | 剧本→分镜脚本 |
| reviewing-micro-drama | `E:\cc wright\.claude\skills\reviewing-micro-drama-scripts\SKILL.md` | 剧本内容安全审核 |
| qiuzhi-skill-creator | `E:\cc wright\.claude\skills\qiuzhi-skill-creator\SKILL.md` | 引导创建自定义 skill |

### Skill 文件格式 (.skill)

```json
{
  "name": "script-analyzer",
  "version": "1.0",
  "description": "从剧本中提取角色、场景、道具信息",
  "trigger_words": ["分析剧本", "提取角色"],
  "system_prompt": "[完整 prompt 文本，即 SKILL.md 的内容]",
  "output_format": "markdown / json / text"
}
```

- 内置 skill 在插件打包时预置
- 用户上传 .skill 文件后解析并存入 chrome.storage.local
- skill 可按文件夹整理

### Skill 栏的文件夹管理

- 用户可创建/重命名/删除文件夹
- 文件夹内可拖拽排序 skill
- 文件夹层级：只支持一级（避免复杂度爆炸）
- skill 文件可从文件夹拖出/拖入

### Skill 栏的删除功能

- 右键菜单或悬停时的 ✕ 按钮
- 删除时确认弹窗
- 删除不可恢复（无回收站）

## 三、交互区（中心栏）

### 工作流程

```
用户上传剧本 (.txt/.doc/.docx)
  → FileReader 读取文本
  → 在交互区显示文本预览
  → 用户从左侧选择/切换 skill
  → 用户点击"执行"
  → LLM 调用 ARK API，按 skill 的 system_prompt 处理
  → 结果展示在交互区
  → 用户：
      ① 手动复制文本 → 粘贴到 jubianai
      ② 点击"保存为 .txt" → 文件存入 skill 栏
```

### 交互区功能

- 文本输入区（可编辑，用户可修改剧本后重新提交）
- LLM 输出展示区（结构化渲染，markdown 格式）
- "执行"按钮 → 调用当前选中 skill + ARK API
- "保存为 .txt" 按钮 → 将输出保存为文件到 skill 栏
- "清空"按钮 → 清空交互区
- 历史记录（上下翻页查看之前的交互结果）

## 四、资产池（右栏）

### 三标签页

| 标签 | 图标 | 默认分类 |
|:----|:----|:---------|
| 角色 | 👤 | 人物类图片 |
| 场景 | 🏠 | 环境类图片 |
| 道具 | 📦 | 物品类图片 |

### 操作

- 拖拽图片到对应标签页的拖拽区 → 自动归入该类
- 点击图片 → 进入画图工具（覆盖层，保留现有 drawing.js）
- 画图编辑后保存 → 副本自动存入同一分类池
- 在分类池中可拖拽调整顺序
- 从分类池拖出图片 → 释放到技能栏 = 删除（确认弹窗）
- 文件夹管理：每个分类内可建子文件夹

### 删除功能

- 悬停图片 → 右上角 ✕ 按钮
- 删除确认弹窗
- 删除不可恢复

## 七、数据存储

| 数据 | 存储位置 | 格式 |
|:----|:---------|:-----|
| Skill 文件内容 | chrome.storage.local | JSON 序列化 |
| 文件夹结构 | chrome.storage.local | JSON |
| 资产图片（缩略图） | IndexedDB | base64 WebP（200px宽，~15KB/张） |
| 资产图片（画图副本） | IndexedDB | base64 PNG |
| 面板位置/大小 | chrome.storage.local | JSON |
| 交互历史 | sessionStorage | JSON 数组 |

### 存储策略

- **chrome.storage.local（~10MB）**：只存文本类数据（skill 文件、文件夹结构、面板配置），占用量很小
- **IndexedDB（无硬上限）**：存图片数据（缩略图 + 画图副本）。浏览器通常允许 Origin 使用硬盘剩余空间的 50% 以上，对图片存储足够
- **sessionStorage**：交互历史，页面刷新不丢，关闭标签页清除

### 图片存储说明

- 用户拖入的原始图片以 File 对象形式存在内存中（会话级），不持久化
- 缩略图（200px 宽 WebP，约 15KB）存入 IndexedDB，用于资产池预览
- 画图保存的副本存入 IndexedDB，用户可随时下载或删除
- 用户关闭标签页后，缩略图和副本仍然保留，下次打开插件时还原

## 八、蜘蛛侠纵横宇宙风格规范

### 配色

| 角色 | 色值 | 用途 |
|:----|:-----|:-----|
| 主背景 | `#0D0D0D` | 画布底色 |
| 面板底色 | `#1A1A1A` | 面板背景 |
| 品红 | `#FF0055` | 主色、按钮、高亮 |
| 电蓝 | `#00E5FF` | 辅色、链接、选中态 |
| 荧光绿 | `#00FF66` | 成功、确认 |
| 柠檬黄 | `#FFEA00` | 警告、提示 |
| 白 | `#FFFFFF` | 主文字 |
| 灰 | `#666666` | 次要文字 |

### 视觉元素

- 面板边框：手绘风格的不规则描边（CSS SVG filter 或 canvas 绘制），2-3px 品红
- 网点背景：面板局部区域使用 Ben-Day dots 纹理（CSS radial-gradient 实现）
- 文字：粗体无衬线字体（Impact / 粗黑体），标题可加描边或 glitch 效果
- 按钮：圆角极小或直角，hover 有 glitch 位移效果（CSS keyframe 抖动）
- 拖拽手柄：漫画速度线风格的条纹
- 滚动条：自定义，品红薄条
- 分界线：锯齿形（zigzag）或喷涂质感

## 九、模型配置

### 内置模型

插件内置一个默认模型端点（ARK API），用户开箱即用。

### 用户自定义代理

用户可添加自己的模型代理，格式：

```json
{
  "url": "https://api.example.com/v1/chat/completions",
  "key": "sk-xxxxx",
  "model": "gpt-4o"
}
```

- 支持多个代理，可在 Skill 执行时选择用哪个
- 代理配置保存在 chrome.storage.local
- 插件执行 LLM 调用时，请求直接发到用户配置的 url，不经过任何中间服务

### 模型选择逻辑

执行 Skill 时，用户可在交互区顶部选择：
1. 使用内置模型（默认）
2. 使用用户添加的某个代理

### 需依赖的 API

| 调用 | 用途 | 备注 |
|:----|:-----|:-----|
| ARK 文本 API | LLM 按 skill 处理剧本（内置） | 现有 background.js 复用 |
| 用户自定义 API | 同上，用户自选 | URL + Key 可配 |
| ARK Vision API | 可选，未来图片自动分类 | 本次不接入，用户手动分类 |
| 无其他外部依赖 | — | 所有逻辑在扩展内完成 |
