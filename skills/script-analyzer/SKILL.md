# 剧本分析 · 资产提示词生成

收到剧本后，按以下三大模块逐一提取并输出。每个模块输出的【图像生成提示词】可直接用于 AI 图像模型生成对应的资产参考图。

---

## 模块一：角色提取

1. 通读剧本，统计所有角色名出现的次数
2. 仅保留**出现超过 9 次**的角色
3. 每个角色按以下格式输出：

```
=== [角色名] ===

【基础信息】
年龄：[X]岁
性别：男/女
人种：东亚人种，中国人
身份：[角色身份描述]

【外形特征】
脸型：[ ]
眉型：[ ]
眼型：[ ]
鼻型：[ ]
唇型：[ ]
发型：[颜色/长度/样式]

【服装（前期）】
外套：[ ]
内搭：[ ]
下装：[ ]
鞋履：[ ]
配饰：[ ]

【服装（后期·如有变化）】
外套：[ ]
内搭：[ ]
下装：[ ]
鞋履：[ ]
配饰：[ ]

【图像生成提示词】
单张图，16:9，左正面半身特写，右三视图，纯白背景，真人影视短剧写实风格，
[年龄]岁[性别]，东亚人种中国人，[脸型][眉型][眼型][鼻型][唇型]，[发型]，
前期服装[外套+内搭+下装]（如有后期则写"后期服装[外套+内搭+下装]"），
摄影级画质，细节清晰，皮肤纹理自然
```

- 前期/后期分两段时，【图像生成提示词】也输出两段（注明"前期"/"后期"）
- 角色不存在前后期变化时，只输出一套，去掉后期相关字段

---

## 模块二：场景提取

1. 列出剧本中所有场景地点及其对应的集号
2. 仅保留**出现超过 1 次**的场景
3. 每个场景按以下格式输出：

```
=== [场景名称] ===

【拍摄参数】
机位高度：[文字描述]约[X]m（如：人眼高度约1.5m / 俯视约2.5m / 仰视约1.0m）
焦距：24mm / 35mm / 50mm / 85mm
画幅：16:9 横屏

【场景类型】
室内 / 室外
时段：清晨 / 正午 / 黄昏 / 夜晚
季节：春季 / 夏季 / 秋季 / 冬季

【空间布局】
场景图左侧：[墙体/门窗/家具/通道等]
场景图正前方：[主要视觉焦点方向]
场景图右侧：[墙体/门窗/家具/通道等]
空间尺寸：宽阔 / 适中 / 狭小
可信度：✅ 从剧本推断 / ⚠️ 需要用户确认

【视觉描述】
空间氛围、光影质感、色调风格、环境情绪。
墙体材质、地面、家具样式、颜色、软装细节。光线如何落在墙面/地面上、空气中的微粒和气味、空间的大小和材质、环境音效、色调和温度感。
**不得出现任何人物**——只写空间本身。

【光源】
主光：[场景图物体]的[光源名]从画面[方位]照入，[视觉效果]
辅光：[场景图物体]的[光源名]从画面[方位]补入，[视觉效果]

【色调与材质】
主色调：[颜色]
材质：[具体材质列表]

【风格】
[风格描述]

【图像生成提示词】
[以上信息的压缩版：场景类型+空间布局+视觉亮点+光源+色调材质+风格+画质，
去掉"场景图左侧/正前方/右侧"标签，恢复为自然方位描述，
一段可直接用于AI图像生成模型的prompt]
```

- 如果剧本中没有足够信息推断空间布局（左/正前/右），在可信度标注「⚠️ 需要用户确认」
- 拍摄参数中焦距优先选择 35mm（空间展示最自然），除非场景特别宽阔（选24mm）或特别狭小（选50mm）

---

## 模块三：道具提取

1. 提取剧本中**反复出现或具有重要功能**的道具
2. 对每个道具标注其出现的场景和功能作用
3. 每个道具按以下格式输出：

```
=== [道具名] ===

【基础信息】
类型：文件/首饰/武器/日用品/其他
材质：[ ]
尺寸：[ ]
颜色：[ ]
出现场景：[场景名]
功能作用：[该道具在剧情中的功能]

【视觉特征】
表面处理：光泽 / 磨砂 / 做旧 / 全新
特殊标识：[字体/LOGO/花纹等]
特定光线下的质感：[反光/透光/哑光]
前后期状态变化：[如有则注明，如无则写"无变化"]

【图像生成提示词】
白底图，产品摄影风格，[道具名]，[材质]，[尺寸]，[颜色]，[表面处理]，
[特殊标识]，摄影级画质，细节清晰，光影干净，产品居中，无人物
```

---

## 输入格式处理

- **.txt 文件**：直接读取全文分析
- **.docx / .pdf 文件**：使用 Read 工具读取内容后分析
- **粘贴文本**：直接对文本进行分析

## 输出格式

以分隔线 `---` 分隔三大模块。每个模块内部的每个资产条目按上述完整格式输出。每个条目末尾必须有【图像生成提示词】字段，供下游生图使用。

## 模块间衔接

- 角色和道具出现在多个场景中时，在【基础信息】中列出关联的场景名称
- 场景输出中的【空间布局】与 shot-script-creator 的「场景图左侧/正前方/右侧」坐标系完全兼容，分镜脚本可直接引用

## 补充说明

- 三个模块可以单独输出：如只提取场景，不输出角色和道具
- 如有无法提取的信息，如实标注"剧本中未提及"，不要编造
